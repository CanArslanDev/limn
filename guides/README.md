# Limn Guides

Extended documentation for the `limn` package. The root `README.md` is the 5-minute tour; these guides go deeper for developers who need the full picture.

## Start here

| Guide                                   | When to read |
| --------------------------------------- | ------------ |
| [Getting started](getting-started.md)   | You just installed Limn. Walks through configuration, your first call, and how to set provider keys. |
| [API surface reference](api-surface.md) | You want the full list of functions on the `ai` namespace, their option shapes, and their return types. |
| [Cookbook](cookbook.md)                 | You know the basics and want copy-paste recipes for common patterns: structured extraction, parallel calls, retries, streaming with backpressure. |

## Reference

The API reference is generated by TypeScript's compiler from the source and shipped as `.d.ts` files alongside the JavaScript build. Editor tooltips show the JSDoc comments inline; a fuller HTML reference will land at `https://limn.dev/api` once the docs site ships.

## Architecture

The root `README.md` has a condensed architecture diagram. For the full picture (layered import rules, provider dispatch, trace pipeline, extension points), see `CONTRIBUTING.md` and the architecture test at `test/architecture/import_flow.test.ts`. The architecture test is the executable specification of how layers import from each other; if it passes, the layering is sound.
